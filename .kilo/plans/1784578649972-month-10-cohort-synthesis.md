# Month 10: Cohort Synthesis, Statistical Validation & Poster Master Canvas

## Context

This plan closes out a multi-month computational oncology project (an anisotropic
PDE tumor-growth pipeline run across an 8-patient cohort, PAT_0000–PAT_0007).
Months 7–9 produced three metric families in `output/`:

- `anisotropic_geometry_metrics.json` — Phase 1: anisotropic tensor growth
- `stromal_feedback_metrics.json` → Phase 2: microenvironment-coupled feedback
- `adaptive_geometry_metrics.json` — Phase 3: MTD vs Adaptive dosing

Month 10 unifies these into a single dataset, runs cohort-wide statistics, renders
a publication-grade 4-panel master PNG, and finalizes the GitHub repo for a
science-fair poster.

## Critical facts discovered during planning (bind the implementation)

1. **Two metric JSON shapes exist.** `anisotropic_geometry_metrics.json` is a
   dict with keys `phase1_tensor_validation`, `phase2_mass_conservation`,
   `phase4_geometry_metrics` (list of 8 patient dicts). The other two files
   (`stromal_feedback_metrics.json`, `adaptive_geometry_metrics.json`) are
   bare top-level lists of 8 patient dicts. Ingestion must handle both.
2. **Df bounds differ per phase.** Phase 1 (anisotropic) `fractal_dimension`
   ranges 1.20–1.55; Phase 2 (stromal) `fractal_dimension` ranges 0.51–0.88
   (uses a different estimator). Apply `[1.0, 2.0]` only to Phase 1; apply
   `[0.4, 1.0]` to Phase 2. Record both bounds in the output JSON.
3. **Front Correlation range is 0.9317–0.9565.** PAT_0004 (0.9317) is below
   the prose 0.95 threshold. Use the rule `front_correlation ≥ 0.90` as the
   hard floor and report the realized minimum in summary text.
4. **Drug reduction range is 63.3%–73.0%** (PAT_0004 lowest, PAT_0000 highest).
   Use the actual `63–73%` range in Panel D and poster copy — NOT `65–73%`.
5. **Honest framing of adaptive therapy (important).** Resistant fraction is
   1.0 in BOTH MTD and Adaptive arms for all 8 patients, and
   `ttp_adaptive ≈ ttp_mtd` (ratio ≈ 1.0). The genuine win is drug-exposure
   reduction, NOT extended time-to-progression or sensitive-clone preservation.
   Frame Adaptive arm as **non-inferior TTP at 63–73% lower drug exposure**.
   The TTP t-test becomes a non-inferiority / equivalence check (report the
   p-value honestly; p>0.05 is expected and supports equivalence).
   Do NOT claim adaptive preserves sensitivity or lengthens TTP.
6. **No spherical/isotropic baseline exists in current outputs.** Script 45
   must synthesize one: reuse the FK solver with
   `D_parallel = D_perpendicular = D_base` (isotropic/spherical diffusion) for
   all 8 patients, compute the same geometry metrics (Df, tract alignment,
   elongation), then run paired t-tests anisotropic-vs-isotropic.
7. **Interpreter is `venv/Scripts/python.exe`** (Python 3.14.4, numpy 2.5.1,
   scipy 1.18.0, matplotlib 3.11.1). `.venv/Scripts/python.exe` is broken
   (no numpy). `run_all.sh` MUST pin to `venv/Scripts/python.exe`
   (or, for cross-platform, source `venv/Scripts/activate`).
   Existing code already uses `np.trapezoid` (numpy 2.5-compatible) — keep
   this convention; do NOT introduce `np.trapz`.
8. **`.gitignore` already excludes** `*.h5ad`, `*.npy`, `*.pt`, `*.joblib`,
   `__pycache__/`, `*.pyc`. It does NOT exclude `*.npz`. Push decision: commit
   source + JSON + PNGs only; add `output/*.npz` to `.gitignore` so cohort
   array artifacts are NOT pushed (keeping the repo lean). JSON + PNGs are the
   reproducible evidence trail; downstream users regenerate .npz by running
   scripts 42–44.

### Step A — Ingestion (Week 1)

`load_all_metrics() -> dict`:
- For each file in `INPUT_FILES`: open, JSON-load.
- Normalize: if file is a dict containing `phase4_geometry_metrics`, treat
  the inner list as the per-patient array; also surface
  `phase1_tensor_validation` and `phase2_mass_conservation` sub-dicts.
  If file is a top-level list, use directly.
- Verify 8 patient IDs present (`PAT_0000`..`PAT_0007`) in each phase.
- Verify expected metric keys exist per record (per-phase schema assertion);
  raise `RuntimeError` listing any missing keys / boundary mismatches.

`validate_ranges(per_phase_records) -> dict`:
- For each phase, scan `fractal_dimension`, `perimeter`, `area`,
  `tract_alignment_fraction`, `front_correlation` (Phase 2 only),
  `drug_reduction` (Phase 3 only), `ttp_mtd`, `ttp_adaptive`.
- Apply ranges: Phase1 Df in `(1.0, 2.0)`, Phase2 Df in `(0.4, 1.0)`,
  `front_correlation >= 0.90`, `drug_reduction in (0, 1)`.
- Produce a per-patient pass/fail table; overall `validation_pass` boolean.
- Print summary table to console.

`build_master_summary() -> dict`:
- Merge to a unified 8-patient cohort JSON:
  ```json
  {
    "schema_version": "month10-v1",
    "generated_at": "<ISO timestamp>",
    "validation": { ... per_phase, per_patient, overall_pass },
    "tensor_validation": { <from aniso phase1_tensor_validation> },
    "mass_conservation": { <from aniso phase2_mass_conservation> },
    "patients": [
      {
        "patient_id": "PAT_0000",
        "phase1_anisotropic": { ... },
        "phase2_stromal":     { ... },
        "phase3_adaptive":    { ... },
        "spherical_baseline": { fractal_dimension, elongation, tract_alignment },
        "anisotropy_excess":  { df_delta, elongation_delta }
      }, ...
    ]
  }
  ```
- Write to `output/master_cohort_summary.json` (indent=2, floats normalized).

### Step A.2 — Spherical/isotropic baseline (D3)

`run_spherical_baseline() -> list[dict]`:
- Reuse FK machinery from script 42/44 by importing the solver classes
  from `src.42_anisotropic_pde` (if importable) OR re-implement a minimal
  isotropic FK step inline (preferred — simpler, no dep cycle) — see below.
- Build isotropic 2×2 tensor: `D_xx = D_yy = ISO_D`, `D_xy = 0` everywhere
  (NO tract anisotropy). The same `AnisotropicFKSolver.step` from script 44
  can be reused with these constant tensors.
- Per patient: same seed conventions as script 44 (`(40, 40)` both clones),
  run `N_STEPS=500`, save final density frame.
- Compute on the final frame (mirror the geometry helpers in script 42):
  - `fractal_dimension` (box-count on thresholded density > 0.1*K)
  - `tract_alignment_fraction` (front-pixel angular alignment to tract —
    will be ~0.5 random for isotropic since no preferred direction)
  - `elongation` (ratio of major/minor PCA axis of thresholded tumor mask)
- Return list of `{patient_id, fractal_dimension, elongation,
  tract_alignment_fraction}`.
- Cache results to `output/isotropic_baseline_metrics.json` so re-runs skip
  the simulation (idempotency anchor; matches D5).

If importing solver classes across scripts is awkward, re-implement a 30-line
isotropic FK step (5-point Laplacian, explicit Euler, clamp to [0, K]) inside
script 45. This is the safer path — keeps script 45 self-contained.

### Step B — Statistical synthesis (Week 2)

`run_statistics(master) -> dict` (use `scipy.stats`):

1. **Anisotropy vs Isotropic (D3-driven, paired):**
   - Paired t-test on `fractal_dimension`: anisotropic per-patient
     (from `phase1_anisotropic.fractal_dimension`) vs isotropic per-patient
     (from `spherical_baseline.fractal_dimension`).
   - Paired t-test on `tract_alignment_fraction` (anisotropic vs ~0.5 baseline).
   - Report: t-statistic, p-value, degrees of freedom, mean difference,
     Cohen's d (effect size). Expected: Df anisotropic significantly higher
     (p<0.001 range plausible with n=8 paired).
2. **MTD vs Adaptive toxicity reduction:**
   - `drug_reduction` array (8 values). Report mean, SD, 95% CI
     (t-based), min, max.
   - Paired t-test on `drug_auc_mtd` vs `drug_auc_adaptive`
     (expected p<0.001 — drug exposure dramatically lower under adaptive).
   - Compute Pearson/Spearman correlation between `mean_inflammation`
     (Phase 3 `mean_inflammation`) and `drug_reduction` — does inflammation
     predict holiday benefit?
3. **TTP non-inferiority (D2 — honest):**
   - Paired t-test on `ttp_mtd` vs `ttp_adaptive`. Report p-value as-is
     (expect p>0.05 → cannot reject equality → SAME TTP, supports
     non-inferiority claim, NOT superiority).
   - State explicitly in console: "Adaptive achieves equal TTP at 63–73%
     lower drug exposure (non-inferiority framing)".
   - Also report `ttp_ratio` mean and SD.
4. **Multi-omic stratification:**
   - Per patient, pull `zone_scores` for `S100A8/S100A11/LST1` (already
     embedded in all three metric files as `zone_scores.Leading Edge` etc.
     — use `mean_inflammation` as scalar summary OR mean across the three
     zone scores).
   - Pearson + Spearman correlation of `mean_inflammation` vs `ttp_mtd`
     and vs `ttp_adaptive`. Test whether higher inflammation predicts
     earlier progression.
   - Define 3 risk tiers by inflammation tertile (Low/Mid/High) and report
     mean TTP per tier.
5. Print summary table to console containing each test's name, statistic,
   p-value, effect size, and a one-line interpretation. Use realistic
   precision (no fake "p<0.001" — print exact p using a formatter like
   `p={p:.4g}`; if p<0.0005 say "p<0.001" inline).

Store the full statistics dict inside `master_cohort_summary.json` under
`"statistics"` (counts as the Week 2 deliverable artifact).

## `run_all.sh` (Week 4 master executor)

```bash
#!/usr/bin/env bash
set -euo pipefail
PY="venv/Scripts/python.exe"
[ -x "$PY" ] || { echo "venv/Scripts/python.exe not found"; exit 1; }
echo "Using interpreter: $PY"
"$PY" --version
"$PY" -c "import numpy, scipy, matplotlib, pandas; print('deps ok')"

run() {
  local script="$1"; shift
  local marker="$2"; shift
  if [ -f "$marker" ]; then
    echo "[skip] $script (marker $marker exists)"
  else
    echo "[run ] $script"
    "$PY" "$script" "$@"
  fi
}

run src/42_anisotropic_pde.py        output/anisotropic_geometry_metrics.json
run src/43_stromal_feedback.py        output/stromal_feedback_metrics.json
run src/44_adaptive_therapy.py        output/adaptive_geometry_metrics.json
run src/45_validation_synthesis.py    output/master_cohort_summary.json

echo "[done] Full Month 7-10 pipeline complete."
```

Notes:
- Bash script on Windows requires Git Bash / WSL. Document in audit log.
  Alternative: a one-line PowerShell equivalent is acceptable if the user
  prefers; flag in Step E audit which shell was used.
- Skip logic only works because each upstream script writes its "marker"
  file on success. If a script failed previously the marker will still
  exist — acceptable for re-runs; full clean rebuild = delete the markers.

## Risks & mitigation

- **Risk:** `anisotropic_evolution_PAT_XXXX.npz` field schema unknown for
  Panel A spatial render. **Mitigation:** at runtime in step C, list npz
  keys with `n.files`; prefer the last-indexed density frame; if absent,
  fall back to a reconstructed contour from the saved geometry dict
  (`perimeter`, `area` → circular patch). Document the fallback in audit log.
- **Risk:** `np.trapezoid` only exists in numpy ≥ 2.0 (we have 2.5.1 ✓).
  **Mitigation:** do not call `np.trapz` anywhere in script 45.
- **Risk:** Spherical baseline (D3) sim could take >30 s for 8 patients ×
  500 steps on a 100×100 grid. **Mitigation:** keep N_STEPS=500 and the
  isotropic step simple (vectorized 5-point Laplacian); cache to
  `output/isotropic_baseline_metrics.json` so `run_all.sh` only pays the
  cost once.
- **Risk:** User may want hay to commit per-patient .npz later. **Mitigation:**
  .gitignore change is reversible; the cache JSON still records the cohort
  summary. Mention unstaged .npz in the audit footer.
- **Risk:** Paired t-test n=8 is low power; p-values may not reach the
  prose's "p<0.001" claim for Df comparison. **Mitigation:** Report actual
  p-values; the stromal/anisotropic effect sizes are large enough that
  reaching p<0.05 is plausible — if a particular test yields p>0.05,
  report it honestly and adjust the `POSTER_KEY_FINDINGS.md` bullet to
  match (no hardcoded `<0.001`).
- **Risk:** Running scripts 42–44 via `run_all.sh` may take many minutes
  each (Months 7–9 originally took ~hours). **Mitigation:** keep the skip
  markers in place so the wrapper is a no-op for upstream phases if their
  marker JSONs exist. Document approximate timings in `MONTH10_AUDIT.md`.
